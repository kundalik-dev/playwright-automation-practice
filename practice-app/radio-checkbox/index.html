Input / Radio / Checkbox — Playwright Practice

Covers locator strategies: id, name, placeholder, label, role, data-testid, aria-*, CSS, XPath, dynamic, and no-stable-locator scenarios.

1. Basic Text Inputs Beginner

Locators to try: page.getByLabel() · page.locator('#id') · page.getByPlaceholder() · page.locator('[name="..."]')

2. Special & Tricky Inputs Intermediate

Locators to try: page.locator('[type="tel"]') · getByRole('spinbutton') · locator('[aria-label="..."]') · CSS attribute selectors

Hidden field present: #csrf-token with value abc123-csrf-secret. Use locator('#csrf-token').inputValue() to read it.

3. Radio Buttons Beginner → Advanced

Locators to try: getByRole('radio', {name:'...'}) · locator('[name="gender"]').filter({hasText:'Male'}) · locator('input[type="radio"][value="..."]') · XPath sibling axis

3a — Gender (id + for label)
3b — Payment Method (data-testid on wrapper)
3c — Subscription Plan (inline / no id — only name+value)
3d — Dynamic Radio (value is generated / unpredictable) Tricky

No stable value attr. Locate by visible text or aria-labelledby.

3e — Star Rating (reverse-order radio trick) Tricky

Inputs are in reverse DOM order (5→1). Use locator('input[name="rating"][value="4"]')

—

4. Checkboxes Beginner → Advanced

Locators to try: getByRole('checkbox', {name:'...'}) · getByLabel() · locator('[data-testid="..."]') · locator('nth=N') · XPath following-sibling::text()

4a — Terms & Conditions (single, required)
4b — User Permissions (group with name attribute)
4c — Select-All Pattern Intermediate

Automate: check "Select All" → verify all others checked. Uncheck one → "Select All" becomes indeterminate.

4d — Notification Preferences (aria-label, no id) Tricky
4e — Toggle Switch (checkbox + CSS, no visible checkbox) Tricky

No type="checkbox" visible in UI. Still a checkbox in DOM. Use getByRole('checkbox', {name:'Dark Mode'})

Dark Mode
4f — Dynamically Loaded Checkboxes Tricky

Items rendered after a delay. Use page.waitForSelector() or expect(locator).toBeVisible() before interacting.

    6. Locator Strategy Cheat Sheet

    Strategy Playwright Syntax Best For Priority
    data-testid getByTestId('...') App with testid attributes ⭐ Highest
    Role + Name getByRole('checkbox', {name:'...'}) Accessible elements ⭐ High
    Label getByLabel('Email') Form fields with visible labels ⭐ High
    Placeholder getByPlaceholder('Search…') Inputs with no label Medium
    Text getByText('Submit') Buttons / links / static text Medium
    CSS selector locator('input[name="gender"]') Attribute-based targeting Medium
    XPath locator('//label[text()="Male"]/input') Parent/sibling traversal Use when CSS fails
    nth locator('input[type="radio"]').nth(2) Dynamic lists, no stable id Last resort
    filter locator('label').filter({hasText:'UPI'}) Narrowing a broad selector Combine with others
    aria-label locator('[aria-label="Phone number"]') Accessible inputs without label Medium
