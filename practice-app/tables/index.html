Tables Practice

Static, sortable, searchable, paginated, row-action, and combined-grid tables — for practicing Playwright table locators and real-world interaction patterns.

1. Basic static table Beginner

Fixed hard-coded data. Practice all locator strategies: getByRole('table') · getByRole('columnheader') · getByRole('row') · getByRole('cell') · nth() · filter({ hasText }) · data-testid. Also practice locating a cell by its column index and by row context.

2. Sortable table Intermediate

Click a column header → sort ascending; click again → descending; third click → unsorted. Assert aria-sort attribute ("ascending" / "descending" / "none") and first-row cell text after each click. Sort by text, number, and date columns.

Text search (by name) + department dropdown filter — both work together. Assert search-result-count text and visible row count after each filter change. Search for a non-existent name to trigger the empty state.

4. Paginated table Intermediate → Advanced

15 employees, 5 per page. Assert: row count per page, aria-current="page" on active button, Prev disabled on page 1, Next disabled on last page, "Showing X–Y of 15" updates after navigation.

5. Table with row actions (Edit / Delete) Intermediate → Advanced

Each row has Edit and Delete buttons. Edit replaces cells with inline inputs; Save commits the change; Cancel reverts. Delete removes the row from the DOM. Practice: scoping buttons by row (getByTestId('actions-row-2').getByRole('button', { name: 'Edit' })), asserting DOM removal, inline input interactions.
