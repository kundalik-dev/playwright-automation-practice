Date Pickers Practice

Native and custom date pickers to practice Playwright locators & interactions. Selections are echoed into output boxes for assertions.

1. Native date input Beginner

Standard <input type="date">. Playwright: use fill('YYYY-MM-DD'). Multiple locator strategies: id, name, label, aria-label, data-testid. Second input has no id — practice label/testid locators.

No date selected

2. Date with constraints Beginner → Intermediate

min / max restrict the calendar UI. Playwright's fill() bypasses the UI but the browser reports validity. Practice: assert min/max attribute values, attempt an out-of-range fill, check validity.rangeOverflow via evaluate().

No constrained date selected

3. Date & time variants Intermediate

Different input types — each needs its own value format in fill(): datetime-local → 'YYYY-MM-DDTHH:MM' | month → 'YYYY-MM' | week → 'YYYY-WNN' | time → 'HH:MM'.

No variant date/time selected

4. Date range picker (native pair) Intermediate

Two native date inputs acting as a start/end range. Check-out is disabled until check-in is set, and its min auto-updates. Practice: fill start → assert end's min attribute equals start → fill end → assert calculated nights → click Book.

No range selected

5. Custom calendar picker Advanced

No native <input type="date">. A div-based calendar (like Flatpickr / React Datepicker / MUI DatePicker). fill() will NOT work — click the input to open the popup, navigate months with prev/next, then click the day cell. Days 5, 13, 20 of the current month are booked (blocked). Past days are disabled.

6. Date filter (table scenario) Advanced

Real-world: filter a data table by a date range. Practice: fill from/to → click Apply → assert visible row count. Leaving a field blank means "no bound on that side". Clear resets all rows. Interview topic: asserting dynamic table row counts after filter changes.
